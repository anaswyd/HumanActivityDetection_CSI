WiFi-Sensing: Personen- und Aktivitätserkennung mittels WLAN-SignalenDieses Projekt demonstriert die Nutzung von Channel State Information (CSI) gewöhnlicher WLAN-Signale zur kontaktlosen Erkennung menschlicher Aktivitäten. Es verbindet modernste Signalverarbeitung mit Deep Learning und zeigt den Transfer von hochpräzisen Labordaten auf kostengünstige IoT-Hardware (ESP32) in realen Wohnumgebungen.  📑 ProjektübersichtIm Gegensatz zu kamerabasierten Systemen bietet WiFi-Sensing eine privatsphäre-wahrende Alternative zur Überwachung im Sicherheits- und Gesundheitswesen. Durch die Analyse physikalischer Kanalveränderungen, die durch menschliche Bewegungen verursacht werden, können Aktivitäten wie Gehen, Sitzen oder Stehen algorithmisch klassifiziert werden.  Technische Kernaspekte:Hybride Modellarchitektur: Entwicklung eines ressourceneffizienten CNN-LSTM-Netzwerks (33.044 Parameter), das räumliche Subcarrier-Topologien mit zeitlicher Bewegungsdynamik kombiniert.  Robustes Signal-Processing: Implementierung einer zweistufigen Filterkaskade aus Hampel-Filtern zur Ausreißereliminierung und Savitzky-Golay-Filtern zur Signalglättung.  Wissenschaftliche Validierung: Fokus auf die Vermeidung von Temporal Data Leakage durch den Einsatz von Group K-Fold Cross-Validation anstelle einfacher Train/Test-Splits.  Echtzeit-Inferenz: Entwicklung einer Live-Pipeline für ESP32-Mikrocontroller inklusive Majority-Vote-Verfahren zur Vorhersagestabilisierung.  🛠 Technologien & MethodenDeep Learning: TensorFlow / Keras (CNN, LSTM, hybride Modelle), GELU-Aktivierung, SMOTE zur Klassenbalancierung.  Signalverarbeitung: Z-Transformation, Data Augmentation (Magnitude Scaling, Time Shifting, Gaußsches Rauschen).  Hardware: ESP32 Mikrocontroller im 20-MHz-Band, serielle Echtzeit-Datenextraktion aus 64 Subcarriern.  📊 Ergebnisse & Key InsightsDas Projekt belegt die signifikante Differenz zwischen idealisierten Laborbedingungen und realen IoT-Deployments:Labor-Baseline: Eine optimierte CNN-Architektur erzielte auf Forschungsdatensätzen eine Genauigkeit von 99,33 % unter Group K-Fold-Validierung.  ESP32 Live-Umgebung: Auf eigenen, stark verrauschten Sensordaten erreichte das hybride Modell im Standard-Split 96,18 %. Die realitätsnahe Group K-Fold-Validierung ergab eine belastbare Generalisierungsleistung von 69,63 %.  Fazit: Die Arbeit zeigt kritisch auf, dass hohe Einzelmetriken aus kontrollierten Datensätzen nur eingeschränkt aussagekräftig sind und eine belastbare Bewertung zwingend die Generalisierungsfähigkeit unter unbekannten Bedingungen berücksichtigen muss.  📁 DokumentationFür tiefergehende Informationen zu den mathematischen Grundlagen, den Filterkaskaden und den vollständigen Evaluationsmetriken stehen folgende Dokumente zur Verfügung:Schriftliche Ausarbeitung (PDF)Präsentationsfolien (PDF)Dieses Projekt wurde im Rahmen des Info-Projekts (WS 25/26) am Fachbereich Informatik und Ingenieurwissenschaften der Frankfurt University of Applied Sciences erstellt.Autoren: Anas Loukili & Yassin Ouchen
